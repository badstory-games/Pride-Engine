# Pride Engine - Контекст проекта

## Описание проекта
Pride Engine - это 2D игровой движок с нуля на чистом JavaScript и WebGL2. 
Проект использует модульную систему ES6 и ECS архитектуру. 
Движок включает в себя полнофункциональный визуальный редактор, работающий в браузере.
Разработка ведется без использования бандлеров и упаковщиков.

## Архитектура ECS

### Entity (Сущность)
**Файл:** `js/core/Entity.js`

Сущность - это контейнер для компонентов. Не содержит собственной логики.
- Уникальный ID (автоинкремент)
- Имя сущности
- Коллекция компонентов (Map)
- Флаг активности
- Методы: `addComponent()`, `getComponent()`, `hasComponent()`, `removeComponent()`

### Component (Компонент)
**Файлы:** `js/components/*.js`

Компоненты хранят только данные. Каждый тип компонента в отдельном файле.

**Активно используемые:**
- **TransformComponent** - позиция, поворот, масштаб
- **RenderComponent** - цвет, размеры, видимость, прозрачность
- **MovementComponent** - скорость, ускорение, трение

**Готовы, но не интегрированы:**
- **PhysicsComponent** - физические свойства (масса, коллайдер, гравитация)
- **AnimationComponent** - спрайтовые анимации (кадры, тайминги)
- **ParticleEmitterComponent** - системы частиц
- **ScriptComponent** - пользовательские скрипты поведения
- **AudioComponent** - звуковые эффекты

### System (Система)
**Файлы:** `js/systems/*.js`, `js/renderer/*.js`, `js/input/*.js`

Системы содержат логику обработки компонентов. Выполняются в игровом цикле.

**Порядок выполнения систем:**
1. **InputSystem** - сбор пользовательского ввода (клавиатура, мышь)
2. **PlayerControlSystem** - обработка ввода для управления сущностями
3. **MovementSystem** - обновление позиций на основе скорости
4. **WebGL2Renderer** - отрисовка всех видимых сущностей

## Визуальный редактор

### Компоненты UI
- **Toolbar** - кнопки управления: ▶ Запустить, ■ Стоп, ⏸ Пауза, + Добавить, 💾 Сохранить, 📂 Загрузить
- **Scene Hierarchy** - список всех сущностей с возможностью выбора и Drag&Drop
- **Inspector** - редактирование свойств компонентов выбранной сущности
- **Console** - логи с временными метками и цветовой индикацией
- **Status Bar** - FPS, количество сущностей, позиция мыши, состояние движка
- **Viewport** - canvas WebGL2 для отображения сцены

### Возможности редактора
- **Создание сущностей** - через меню с выбором типа (пустая, спрайт, игрок)
- **Drag&Drop** - перетаскивание сущностей мышью в режиме редактирования
- **Редактирование свойств** - изменение параметров компонентов через инспектор
- **Горячие клавиши:**
  - `Ctrl+S` - сохранить сцену
  - `Delete` - удалить выбранную сущность
  - `N` - создать новую сущность
  - `WASD` / `Стрелки` - управление игроком (в режиме игры)
- **Сохранение/загрузка** - сцены сохраняются в localStorage в формате JSON
- **Фокусировка** - двойной клик по сущности в иерархии
- **Адаптивный canvas** - автоматически подстраивается под размер окна

## Игровой цикл

### Этапы выполнения каждого кадра:
1. Вычисление deltaTime (в секундах)
2. Обновление всех систем (если не пауза):
   - Сброс состояний ввода одного кадра
   - Обработка пользовательского ввода
   - Обновление физики и движения
3. Рендеринг сцены:
   - Очистка буфера
   - Сортировка сущностей по zIndex
   - Отрисовка через WebGL2 шейдеры

### Производительность:
- Использование `requestAnimationFrame` для оптимального рендеринга
- DeltaTime для независимости от частоты кадров
- WebGL2 шейдеры выполняются на GPU
- Фильтрация только активных и видимых сущностей

## WebGL2 Renderer

### Технические характеристики:
- **Шейдеры:** Вершинный и фрагментный на GLSL ES 3.0
- **Трансформации:** На GPU (позиция, поворот, масштаб)
- **Поддержка цветов:** HEX (#RRGGBB), RGB/RGBA
- **Прозрачность:** Поддержка opacity через uniform
- **Z-сортировка:** По zIndex компонента RenderComponent
- **Буферизация:** Единый буфер вершин для всех прямоугольников
- **Компиляция:** Ленивая инициализация при первом рендере

### Uniform переменные:
- `u_resolution` - разрешение canvas
- `u_translation` - позиция сущности
- `u_scale` - размеры с учетом масштаба
- `u_rotation` - угол поворота
- `u_color` - цвет заливки
- `u_opacity` - прозрачность

## Система ввода

### Клавиатура:
- Отслеживание состояния клавиш (нажата/отпущена)
- Разделение на удержание, нажатие в кадре, отпускание в кадре
- Поддержка любых клавиш через `event.key`

### Мышь:
- Позиция в координатах canvas и мировых координатах
- Состояние кнопок (нажата/отпущена)
- Определение клика в текущем кадре
- Предотвращение контекстного меню на canvas
