# Pride Engine - Контекст проекта

## Описание проекта
Pride Engine - это 2D игровой движок с нуля на чистом JavaScript и WebGL2. 
Проект использует модульную систему ES6 и ECS архитектуру. 
Движок включает в себя полнофункциональный визуальный редактор, работающий в браузере.

## Архитектура

### ECS (Entity-Component-System)
- **Entity**: Контейнер для компонентов, имеет уникальный ID. Файл: `js/core/Entity.js`
- **Component**: Хранит данные. Реализованы: `TransformComponent`, `RenderComponent`. Файл: `js/core/Component.js`
- **System**: Содержит логику обработки компонентов. Файл: `js/core/System.js`

### Визуальный редактор
- Панель инструментов с кнопками управления (Play, Stop, Pause, Add Entity, Save, Load)
- Иерархия сцены - список всех сущностей с возможностью выбора
- Инспектор свойств - просмотр компонентов выбранной сущности и их свойств
- Консоль для логов с временными метками
- Viewport для отображения сцены через WebGL2 canvas
- Адаптивный размер canvas при изменении окна

## Технические детали

### Сборка и запуск
- Проект не требует сборки или бандлеров
- Запускается открытием `index.html` в браузере
- Требуется поддержка ES6 модулей и WebGL2
- Все зависимости загружаются через нативные import/export

### WebGL2
- Контекст создается с базовыми настройками (blending, clear color)
- Вершинный шейдер обрабатывает трансформации на GPU
- Фрагментный шейдер поддерживает uniform цвета
- Буферы вершин создаются для каждого типа геометрии

### Производительность
- Минимальное количество вызовов WebGL (batch rendering пока не реализован)
- Трансформации вычисляются в шейдерах
- Игровой цикл использует requestAnimationFrame
- Delta time вычисляется в секундах для физики

### Хранение данных
- Сцены сохраняются в localStorage в JSON формате
- Компоненты сериализуются с сохранением типов
- При загрузке компоненты воссоздаются через конструкторы

## Заметки по разработке

### Принципы
- Чистый JavaScript без зависимостей
- ECS архитектура для гибкости и производительности
- Компоненты - чистые данные, системы - чистая логика
- Документирование через JSDoc
- Модульность и переиспользование кода

### Стиль кода
- ES6 классы с наследованием
- JSDoc для всех публичных методов и свойств
- Приватные методы с префиксом `_` (планируется переход на #)
- Константы в UPPER_CASE
- Именование: camelCase для методов и переменных, PascalCase для классов

### Процесс разработки
- Каждое изменение документируется в этом файле
- Новые функции сначала проектируются, затем реализуются
- Тестирование в браузере после каждого изменения
- Рефакторинг при необходимости с сохранением обратной совместимости

### Известные проблемы
- Canvas не масштабируется идеально при ресайзе
- Нет обработки ошибок шейдеров в production режиме
- Отсутствует валидация входных данных в редакторе
- Сохранение только в localStorage (нет экспорта в файл)
